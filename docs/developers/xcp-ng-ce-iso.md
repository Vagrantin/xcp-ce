## Purpose

This repo takes the signed `xo-lite-community` RPM produced by
[`xolite-ce`](xolite-ce) and the signed `xoa-proxy` RPM produced by
[`xoa-proxy`](xoa-proxy), overlays them onto a stock XCP-ng 8.3 base using the
official [`create-install-image`](https://github.com/xcp-ng/create-install-image)
toolchain, and publishes the resulting ISO as a GitHub Release.

---

## Toolchain — create-install-image

XCP-ng's official installer ISO is assembled with the `create-install-image`
toolchain. XCP-ng CE uses it directly rather than maintaining a fork.

The toolchain provides two scripts:

| Script | Runs as | Produces |
|---|---|---|
| `create-installimg.sh` | **root** | `install.img` (SquashFS ramdisk) |
| `create-iso.sh` | non-root | final `.iso` file |

They must be run separately, in order. `create-iso.sh` accepts a
`--sign-script` argument for MD5 imprinting.

## Repo structure

```
xcp-ng-ce-iso/
├── configs/
│   ├── base/
│   │   └── CUSTOMREPO.tmpl     ← community repo template
│   └── 8.3/                    ← XCP-ng 8.3 config (target)
├── community-repo/
│   └── x86_64/
│       └── repodata/           ← generated by createrepo_c
├── scripts/
│   └── debug/                  ← debug helper scripts (overlaid in CI)
├── xcp-ng-ce-public.asc        ← community GPG public key (both subkeys)
├── Dockerfile.build             ← build environment
└── .github/workflows/
    └── build-iso.yml
```

### CUSTOMREPO.tmpl

The community repo definition template must be copied from `configs/base/`
to `configs/8.3/` before the build runs:

```bash
cp configs/base/CUSTOMREPO.tmpl configs/8.3/CUSTOMREPO.tmpl
```

This file tells the installer where to find the community RPM repo. The
`--define-repo` flag passes it to both build scripts.

### Community RPM repository layout

`yum` automatically appends the host architecture (`x86_64/`) to the base
URL configured in a repo file. Therefore the community repo must be structured
with `repodata/` under the arch subdirectory:

```
community-repo/
└── x86_64/
    ├── xo-lite-community-*.rpm
    ├── xoa-proxy-*.rpm
    └── repodata/
        ├── repomd.xml
        └── ...
```

Generate the repo metadata against the arch subdirectory:

```bash
createrepo_c community-repo/x86_64/
```

## Build process — step by step

### 1. Prepare the community repo

```bash
# Download signed RPM from xolite-ce latest release
gh release download --repo Vagrantin/xolite-ce \
    --pattern "xo-lite-community-*.rpm" \
    --dir community-repo/x86_64/

# Download signed RPM from xoa-proxy latest release
gh release download --repo Vagrantin/xoa-proxy \
    --pattern "xoa-proxy-*.rpm" \
    --dir community-repo/x86_64/

# Generate repo metadata
createrepo_c community-repo/x86_64/
```

### 2. Inject the community public key into the installer chroot

The ISO signing subkey (`GPG_PRIVATE_KEY` in this repo) is imported into the
runner's keyring at the start of the pipeline. The **public** half is then
exported from that keyring and injected into the installer chroot templates,
so the installer can verify the community RPMs during installation without any
manual key import by the end user.

{: .important }
`rpm --import` inside a container writes to the **container's** RPM database,
not to the chroot installroot that ends up in `install.img`. Always pass
`--root` to target the correct location.

```bash
# Export the public key from the runner keyring
gpg --armor --export "${GPG_KEY_ID}" > /tmp/RPM-GPG-KEY-xcp-ng-ce

# Copy into the toolchain's installroot template
mkdir -p create-install-image/templates/installimg/base/etc/pki/rpm-gpg/
cp /tmp/RPM-GPG-KEY-xcp-ng-ce \
   create-install-image/templates/installimg/base/etc/pki/rpm-gpg/

# Import into the installer chroot at build time
rpm --root="${ROOTFS}" --import /etc/pki/rpm-gpg/RPM-GPG-KEY-xcp-ng-ce
```

### 3. Run create-installimg.sh (root)

```bash
# Must run as root inside the build container
sudo ./create-installimg.sh \
    --define-repo base \
    --define-repo updates \
    --define-repo community \
    [other options]
```

The `--define-repo` flag must list all three repos: `base`, `updates`, and
`community`.

### 4. Run create-iso.sh (non-root)

```bash
./create-iso.sh \
    --sign-script implantisomd5 \
    --define-repo base \
    --define-repo updates \
    --define-repo community \
    [other options]
```

### 5. isohybrid post-processing

{: .important }
This step is **required** for the ISO to boot on physical hardware that doesn't support UEFI.

```bash
isohybrid --uefi output.iso
```

Verify the result with:

```bash
fdisk -l output.iso
xorriso -report_el_torito output.iso
```

### 6. Checksum and publish

The ISO checksum file is signed with the **ISO signing subkey**, which is the
key material stored in `GPG_PRIVATE_KEY` in this repository. This is a
**different subkey** from the one used to sign the RPMs in `xolite-ce` and
`xoa-proxy` — they share the same secret name but hold different private key
material.

```bash
sha256sum output.iso > SHA256SUMS

# Sign the checksum with the ISO signing subkey
echo "${GPG_PASSPHRASE}" | gpg --passphrase-fd 0 --batch \
    --pinentry-mode loopback --yes \
    --detach-sign --armor SHA256SUMS
```

This produces `SHA256SUMS.asc` — a detached signature file that lets anyone
confirm the ISO came from this project without booting it first.

The public key (`xcp-ng-ce-public.asc`) contains the public halves of both
subkeys (RPM and ISO). It is published alongside the ISO so end users can
import it and verify:

```bash
# Option 1 — fetch from keyserver
gpg --keyserver keys.openpgp.org --recv-keys 2F591DB9D2C128C4C3D963F46DA00DCA5BBA215A

# Option 2 — import from the release page
gpg --import xcp-ng-ce-public.asc

# Verify ISO checksum signature
gpg --verify SHA256SUMS.asc SHA256SUMS

# Verify ISO integrity
sha256sum -c SHA256SUMS
```

```bash
# Publish ISO, checksum, signature, and public key
gh release upload <tag> \
    output.iso \
    SHA256SUMS \
    SHA256SUMS.asc \
    xcp-ng-ce-public.asc
```

## install.img — SquashFS internals

{: .warning }
`install.img` is a **SquashFS** archive. Do **not** use `cpio` to repack it.

### Unpacking and repacking

```bash
# Unpack
unsquashfs -d installimg-root install.img

# Make changes inside installimg-root/
# ...

# Repack (must match upstream compression)
mksquashfs installimg-root install.img.new -comp xz -b 131072
```

### answerfile.xml injection (Optional)

`answerfile.xml` referenced via `file:///` in the installer resolves to the
**ramdisk root** (from `install.img`), not the ISO CD-ROM root. To inject an
answerfile for unattended installs, place it inside `install.img` via the
SquashFS repack workflow above.

Disable GPG-related checks via answerfile attributes:

```xml
<installation gpgcheck="false" repo-gpgcheck="false">
  ...
</installation>
```

## CI workflow (GitHub Actions)

The workflow triggers on push to `main` and on workflow dispatch.

Key environment requirements:

```yaml
env:
  TMPDIR: /tmp       # Must be set here AND in Dockerfile
  HOME: /tmp         # Required for non-root create-iso.sh step
```

Key steps:

```yaml
- name: Import ISO signing key
  # GPG_PRIVATE_KEY in this repo holds the ISO signing subkey —
  # different key material from GPG_PRIVATE_KEY in xolite-ce / xoa-proxy
  run: |
    echo "${{ secrets.GPG_PRIVATE_KEY }}" | gpg --batch \
      --pinentry-mode loopback \
      --passphrase "${{ secrets.GPG_PASSPHRASE }}" --import

- name: Export public key for installer chroot
  run: |
    gpg --armor --export "${GPG_KEY_ID}" > /tmp/RPM-GPG-KEY-xcp-ng-ce
    cp /tmp/RPM-GPG-KEY-xcp-ng-ce \
       create-install-image/templates/installimg/base/etc/pki/rpm-gpg/

- name: Set up Docker build environment
  run: docker build -t xcp-ng-build-ready -f Dockerfile.build .

- name: Download xolite-ce RPM
  run: |
    gh release download --repo Vagrantin/xolite-ce \
      --pattern "xo-lite-community-*.rpm" \
      --dir community-repo/x86_64/

- name: Download xoa-proxy RPM
  run: |
    gh release download --repo Vagrantin/xoa-proxy \
      --pattern "xoa-proxy-*.rpm" \
      --dir community-repo/x86_64/

- name: Prepare community repo
  run: createrepo_c community-repo/x86_64/

- name: Copy CUSTOMREPO.tmpl
  run: cp configs/base/CUSTOMREPO.tmpl configs/8.3/

- name: Run create-installimg.sh (root)
  run: |
    docker run --rm --privileged \
      -v $PWD:/build \
      -v $PWD/create-install-image:/create-install-image \
      xcp-ng-build-ready \
      bash -c "cd /create-install-image && sudo ./create-installimg.sh ..."

- name: Run create-iso.sh (non-root)
  run: |
    docker run --rm \
      -e HOME=/tmp \
      -v $PWD:/build \
      xcp-ng-build-ready \
      bash -c "cd /create-install-image && ./create-iso.sh ..."

- name: isohybrid post-processing
  run: isohybrid --uefi output.iso

- name: implantisomd5
  run: implantisomd5 output.iso

- name: Sign checksum
  run: |
    sha256sum output.iso > SHA256SUMS
    echo "${{ secrets.GPG_PASSPHRASE }}" | gpg --passphrase-fd 0 --batch \
      --pinentry-mode loopback --yes \
      --detach-sign --armor SHA256SUMS

- name: Publish to xcp-ce releases
  uses: softprops/action-gh-release@v1
  with:
    repository: Vagrantin/xcp-ce
    files: |
      output.iso
      SHA256SUMS
      SHA256SUMS.asc
      xcp-ng-ce-public.asc
```

{: .note }
The Docker volume mount must reference `create-install-image`.
Verify the mount target matches the actual cloned directory name.
